2. Opsi Library dari Internet

Hasil pencarian library populer 2026, ditampilkan dengan tema aplikasi Anda.

SweetAlert2 asli · live

Modal konfirmasi/alert/input paling populer & cantik. Cocok khusus untuk dialog konfirmasi blocking.

~15 KB gz. Dialog: ✔ · Toast: ✔ · situs. Untuk Vue 3 panggil lib inti langsung.

vue-sonner simulasi bertema

Port dari Sonner (React). Toast minimalis & bersih, muncul dari kanan-bawah, tumpukan halus.

~5 KB gz. Dialog: hanya tombol aksi inline · repo.

Notivue simulasi bertema

Native Vue 3, 5 tema bawaan (di sini gaya "Material"), ikon lingkaran solid, aksesibel + RTL.

~7 KB gz. Paling fleksibel (headless) · situs.

Vue Toastification simulasi bertema

Matang & TypeScript-first. Ciri khas: latar penuh berwarna sesuai tipe, dengan progress bar.

~10 KB gz. Bisa komponen kustom sebagai isi · repo.

vue3-toastify simulasi bertema

Gaya familiar ala react-toastify: kartu putih, aksen kiri berwarna, progress bar bawah.

~12 KB gz. Setup ringkas, auto-container · situs.

Bagaimana dengan framework UI penuh (PrimeVue / Element Plus)?

Keduanya punya komponen Toast + ConfirmDialog sekaligus dan sangat lengkap. Namun tidak disarankan untuk kasus ini: aplikasi Anda memakai CSS tulis tangan tanpa framework, sehingga menambah framework penuh (ratusan KB) hanya untuk pop-up = overkill, memberatkan bundle, dan gaya bawaannya berbeda dari tema Anda (butuh override besar). Cukup dipertimbangkan bila ke depan Anda ingin migrasi seluruh UI ke design-system jadi.

Rekomendasi saya

Pertahankan & percantik sistem buatan sendiri (Section 1) lalu pasang di semua ~12 modul yang masih pakai alert()/confirm() bawaan browser. Alasan: sudah ada di kode, 100% cocok tema, tanpa beban bundle. Jika Anda tetap ingin nuansa library: SweetAlert2 untuk dialog konfirmasi, atau vue-sonner/Notivue untuk toast — tapi tetap butuh di-theme agar tidak bentrok.
